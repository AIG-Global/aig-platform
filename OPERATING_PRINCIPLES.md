## The Daily Question

Every day starts with one question:

> **What is the single most important thing that moves AIGINVEST forward today?**

Not ten things.  
Not five things.  
**One thing.**

When it's finished, we move to the next.

---

## The Product North Star

**Print this. Put it at the top of every roadmap. Refer to it every day.**

> **Build the AI company people trust first. The phone, AIOS, and every service exist to strengthen the AIGINVEST ecosystem.**

This sentence captures everything:

- **AIGINVEST** is the company and ecosystem (supreme)
- **Diana** is the experience (universal interface)
- **AIOS** is the technology (operating system)
- **North Star ONE** is the flagship device (premium hardware)

**Every decision flows from this hierarchy.**

If something doesn't strengthen AIGINVEST, it waits.

---

## How We Measure Progress

### Old Way (Wrong)
"How many features did we build?"

### New Way (Right)
"How many complete user journeys did we finish?"

A complete user journey means:
- User can accomplish something meaningful
- From start to finish without confusion
- Repeatable without manual
- They want to come back

**Quality of journeys > Quantity of features**

---

## The First User Journey: "Meet Diana"

### Definition
A new user arrives at AIGINVEST with no prior context and accomplishes something meaningful in one session.

### Success Criteria

**A new user needs:**
- [ ] No manual
- [ ] No onboarding video
- [ ] No confusion

**A new user understands:**
- [ ] Who AIGINVEST is
- [ ] Why they should stay
- [ ] What Diana can help with

**A new user accomplishes:**
- [ ] Something they came to do
- [ ] Something they're proud of
- [ ] Something worth returning for

**A new user feels:**
- [ ] Welcome
- [ ] Understood
- [ ] That they made the right choice

## AIGINVEST Alpha 0.1: Meet Diana

### Official Milestone Definition

A complete, polished first experience that demonstrates the vision through a working product.

### What Users Can Do

1. **Visit AIGINVEST** — Homepage answers three questions beautifully
2. **Create an account** — Email/password, 90 seconds, smooth
3. **Meet Diana** — Personal greeting, clear invitation
4. **Start a conversation** — Natural, streaming responses
5. **Create a project** — Diana understands intent, creates structure
6. **Generate a document** — User-initiated, saved automatically
7. **Return later** — Everything restored, Diana remembers context

### Definition of "Complete"

All seven items work flawlessly.  
All "Meet Diana" journey criteria met.  
Zero errors.  
Production quality.  

### Success Looks Like

User says: **"This is different. I want to come back."**

## Disciplined Scope Management

### The Commitment

From this point on, founder will be deliberately disciplined about scope.

If a new idea is brilliant but doesn't move Alpha 0.1 forward, it goes in the backlog.

**Reasons to park an idea:**
- It's not part of the current milestone
- It doesn't support "Meet Diana" journey
- It adds complexity to the critical path
- It can wait for Alpha 0.2 or later

**Reasons to build it:**
- It's part of Alpha 0.1 definition
- It strengthens AIGINVEST ecosystem
- It improves the "Meet Diana" experience
- It's essential for the first milestone

## Decision Framework (Unchanged from Constitution)

Every feature still answers four questions:

1. Does it strengthen AIGINVEST?
2. Does it make Diana more helpful?
3. Does it improve North Star ONE?
4. Does it fit AIOS architecture?

**And now:** Is it part of Alpha 0.1's "Meet Diana" journey?

If NO to Alpha 0.1 → Goes to backlog, regardless of four questions.

---

## The Founder's Discipline

### If a brilliant idea emerges:

**Founder asks:**
1. "Does this move Alpha 0.1 forward?"
2. "Does this risk slowing down release?"
3. "Can this wait for Alpha 0.2?"

**If answers are YES, YES, YES:**
- "Park it. We're shipping Alpha 0.1 first."
- "Document it. Reference it in backlog."
- "We'll build it in the next milestone."

**If answer is NO to any:**
- "We need this. How do we add it without scope creep?"
- "Can we do a simpler version for Alpha 0.1?"
- "Who moves to help build this?"
