# AIG Me — personal relationship tracker

A personal CRM for the people in your life: birthdays, custom dates,
relationship notes, manually-tagged photos, and a private log of online
mentions you've found yourself. Same account pattern as AIG Investor
Alerts (local stand-in for a future real backend), same visual language.

## What this deliberately does NOT do

- **No web search.** AIG Me never looks anyone up online — not you, not
  your contacts. Every piece of data here is something you typed or
  uploaded.
- **No facial recognition.** Group-photo tagging is fully manual: you draw
  a box around a person and type their name yourself. Nothing is detected
  or matched automatically.
- **No automated "rating" of people.** The sentiment field on a contact is
  your own private note to yourself ("going well" / "neutral" / "strained")
  — not a score the app computes.
- **No internet-mentions aggregator.** "Mentions" is a manual log: when you
  find something about yourself via a Google Alert, a search you ran, or
  someone telling you, you log it here so it's all in one place. The app
  never performs that search on your behalf — see `monitorList` in
  `app.js` for the self-monitoring nudges, which only link out to tools
  (Google Alerts, Google Images) the person runs themselves.

These aren't arbitrary feature cuts — aggregating personal data about
people from across the web without their direct involvement is the core
mechanic of stalkerware and people-search/OSINT tools, and that's true
regardless of whose name is being looked up.

## Pages / sections (single-page app)

- **Coming up** — birthdays, custom dates, and Mother's/Father's Day (real
  date math: 2nd Sunday of May / 3rd Sunday of June, computed locally —
  edit `mothersDay()` / `fathersDay()` in `app.js` if your locale differs)
  for everyone in your contact list, sorted by days-until.
- **Your people** — contact cards with relationship type, notes, sentiment,
  and a manual photo gallery per person.
- **Tag a group photo** — upload a photo, draw boxes by hand, type each
  person's name. If the name matches an existing contact, the photo gets
  linked to their gallery automatically (still just string matching on a
  name you typed — no recognition).
- **Check what's out there about you** — links out to Google Alerts and
  Google Images reverse search, with a 90-day snooze/reminder cycle.
- **Mentions you've found** — the manual log described above, filterable
  by tone (positive/neutral/concerning), sorted newest first.

## Storage

Everything — contacts, photos (as data URLs), mentions, monitor snooze
state — is stored in `localStorage`, namespaced per logged-in account via
`AIGMeAuth.scopedKey()`. Photos are stored inline as base64 data URLs,
which is simple but not space-efficient; for a real backend, photos should
move to actual file storage rather than living in localStorage strings.

## Mobile

Tested at a 390px-wide touch viewport: sign-in, registration, the full
dashboard, the contact form, and the mention-logging form all render
correctly with no horizontal overflow. The group-photo tagging canvas
supports both mouse drag and touch drag (verified with a simulated touch
sequence) — drawing a box around someone works the same way on a phone as
it does with a mouse.

## Sign-in screen background

An original, procedurally-generated circuit-trace face graphic — a side
profile traced as a glowing circuit-board path (warm amber rim, like light
catching an edge), with branching blue data-lines and junction nodes
through the interior, dissolving into a scattered field of amber/blue/
violet particles. Generated by a small Python script (a hand-defined
profile point path, "jogged" into circuit-style segments, then branch
lines and particles scattered with weighted randomness) — not traced or
copied from any existing photo, stock image, or artwork.

## Files

```
index.html   structure, styling, layout, auth-screen bootstrap script
auth.js      account registration/login/logout — same pattern as the
             AIG Investor Alerts app, swappable for a real backend later
app.js       contacts, dates, photo tagging, mentions log, all rendering
```

## Run it

No build step. From this folder:
```
python3 -m http.server 8000
```
Then open `http://localhost:8000`.
