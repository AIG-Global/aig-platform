This is a sample AIG page

This page represents any normal page on the AIG site — a marketing page, a dashboard, anything. The AIG Ask widget is included with two script tags at the bottom of the page and otherwise needs nothing else: no extra markup, no container div to create yourself.

This is the integration code, exactly as it appears at the bottom of this file:

<script src="core.js"></script>
<script src="widget.js"></script>

Click the floating icon in the bottom-right corner to try it. The widget is currently configured for "own-key" mode — if you haven't set a key in this demo before, it'll ask for one before you can chat (same setup as the standalone AIG Ask page).

To run this widget in "backend" mode instead (once a real backend exists, so visitors don't need their own API key), add a config block before the two script tags: 

<script>
  window.AIGAskWidgetConfig = {
    mode: "backend",
    backendUrl: "https://your-backend.example.com/api/chat"
  };
</script>
<script src="core.js"></script>
<script src="widget.js"></script>

Nothing else about the widget changes — visitors never see a key prompt in that mode, since the backend would hold the shared key instead.
