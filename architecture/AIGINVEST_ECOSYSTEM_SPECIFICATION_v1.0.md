## Part 1: The Two-Account System

### 1.1 Account Architecture

Every AIG member has two distinct, interconnected accounts:

**CASH ACCOUNT (EUR-Based, Fiat)**
```
Purpose:        Bridge to traditional banking
Currency:       EUR (Euro)
Exchange Rate:  1.0 EUR = 1.0 EUR (guaranteed, fixed)
Input Method:   Bank transfer, card deposit, wire transfer
Output Method:  Withdrawal to bank account

Used For:
├─ Deposits from external sources
├─ Paying membership fees
├─ Paying for optional investments
├─ Paying for physical product purchases
├─ Receiving fiat withdrawals
└─ Baseline balance guarantee

Characteristics:
├─ Guaranteed value (never fluctuates)
├─ Fiat currency (real money)
├─ Cross-border compatible
├─ Bank-connected
└─ Used by financial systems
```

**AIG CASH ACCOUNT (Ecosystem-Based, Digital)**
```
Purpose:        Internal ecosystem value circulation
Currency:       AIG$ (AIG Tokens)
Exchange Rate:  Dynamic (market-driven, supply/demand)
Input Method:   Earnings, commissions, rewards
Output Method:  Internal trading platform (peer-to-peer EUR conversion)

Used For:
├─ WDM marketplace purchases (primary use)
├─ App subscriptions (if applicable)
├─ Membership upgrades (if applicable)
├─ Investment opportunities (if applicable)
├─ Receiving affiliate commissions
├─ Receiving achievement rewards
└─ All internal ecosystem spending

Characteristics:
├─ Value fluctuates (market-based)
├─ Digital ecosystem tokens
├─ Cannot leave the ecosystem directly
├─ Converted to EUR only via peer trading
├─ Encourages ecosystem participation
└─ Self-sustaining economic engine
```

### 1.2 The 80/20 Earnings Distribution

**EVERY earnings event in the ecosystem is split 80/20:**

```
When a commission is earned:
├─ 80% → Cash Account (EUR equivalent)
├─ 20% → AIG Cash Account (AIG$ tokens)

Example: €100 commission earned
├─ €80 → Cash Account (immediate, guaranteed)
├─ €20 → AIG Cash Account (~20 AIG$ at 1.0 rate, dynamic from there)

This applies to:
├─ Affiliate/referral commissions
├─ WDM marketplace commissions
├─ Achievement bonuses
├─ Rewards
├─ Any income-generating activity
└─ EXCEPT: Membership fees (paid from Cash Account directly)
```

**Why this matters:**

```
Traditional Model:
├─ Earn money → Keep in bank
├─ Leave ecosystem to spend
├─ Value exits community
└─ Platform loses network effects

AIG Ecosystem Model:
├─ Earn money → 80% Cash (secure), 20% AIG$ (utility)
├─ 80% can be withdrawn if desired (control)
├─ 20% stays in ecosystem encouraging participation
├─ Creates natural incentive: Use AIG$ on WDM (no conversion fee)
├─ Every purchase strengthens network
└─ Value circulates, grows, multiplies
```

### 1.3 Currency Conversion

**From EUR (Cash Account) to AIG$ (AIG Cash Account):**
```
Method: Manual conversion (click button)
Process:
├─ Member initiates conversion
├─ Chooses amount of EUR to convert
├─ System applies current market rate
├─ EUR deducted from Cash Account
├─ AIG$ credited to AIG Cash Account
├─ Conversion fee: [TBD - typically 1-3%]
└─ Instant execution

Why convert?
├─ More purchasing power on WDM (some items priced in AIG$)
├─ Avoid conversion fees on purchases
├─ Participate in ecosystem economy
├─ Support community purchasing power
```

**From AIG$ (AIG Cash Account) to EUR (Cash Account):**
```
Method: Internal Peer Trading Platform
Process:
├─ Member lists AIG$ for sale on internal marketplace
├─ Other members buy at market rate
├─ EUR transferred from buyer to seller
├─ AIG$ transferred from seller to buyer
├─ Platform facilitation fee: [TBD]
└─ No forced conversion

Why this mechanism?
├─ Creates market-based pricing
├─ Prevents artificial inflation/deflation
├─ Encourages ecosystem participation (hold AIG$, don't convert)
├─ Fair price discovery
├─ No platform manipulation of rates
└─ Member-driven economics
```

   ├─ Purchases made in AIG$ preferred

   ├─ Every vendor commission → back to affiliate network

   ├─ All suppliers get same terms (fair)

   └─ Creates virtuous cycle

3. LUXURY + BUSINESS FOCUS
   ├─ Premium goods (vehicles, jewelry, art)
