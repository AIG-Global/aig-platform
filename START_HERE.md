## The 15-Minute Test

**Alpha is successful when:**

A new user can install Diana and within 15 minutes:
1. Meet Diana
2. Ask her something meaningful
3. Get a helpful response
4. Create something from that response (document, task, note)
5. Feel like Diana genuinely understands them

If we can do that, we've won.

## How We Think

### About Diana
- Diana is a companion, not a tool
- Diana learns from every interaction
- Diana is honest about limitations
- Diana gets smarter over time
- Diana earns trust through consistency

### About Development
- Every sprint improves Diana's core capability
- Quality > Speed (always)
- Ship frequently but only when ready
- Users > Metrics
- Long-term vision > Quarterly results

### About the Team
- No politics, direct communication
- Async first, synchronous for decisions
- Celebrate wins, learn from failures
- Everyone's ideas matter
- We're building for 10 years, not 10 weeks

---

## Decision Framework

**When you're unsure what to do, ask:**

1. Does this align with Diana being a trusted companion?
2. Does this improve Diana's core capability?
3. Would users notice if we shipped this?
4. Is this aligned with "Build platform. Teach Diana. Delight user"?

**If 3+ yes → Do it**
**If 2 yes → Discuss with team**
**If < 2 yes → Probably skip it**

## Questions You Should Ask

### Strategic Questions
- "How does this feature teach Diana something new?"
- "Are we still aligned with our north star?"
- "Is this really invisible to the user?"

### Technical Questions
- "What's the simplest way to implement this?"
- "How do we scale this to 1M users?"
- "How do we test this properly?"

### Product Questions
- "Would users notice if we didn't ship this?"
- "Does this improve the 15-minute test?"
- "How does this teach Diana?"

### Team Questions
- "How can I help ship this faster?"
- "What do you need from me?"
- "Did I understand that correctly?"

## Communication Norms

### Async First
- Use written communication by default
- Synchronous meetings only when you need to make decisions together
- Response time: 24 hours during business hours

### Direct & Respectful
- Say what you think, not what you think people want to hear
- Disagree openly, commit fully
- No politics or posturing

### Documentation First
- Document decisions and why you made them
- Document failures and what you learned
- This becomes knowledge for the team

### Celebrate Wins
- Every shipped feature matters
- Every bug fix matters
- Every person matters
- We celebrate together
