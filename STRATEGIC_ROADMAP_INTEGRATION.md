## The Guiding Principle (Refined)

Leadership provided this refined principle:

> **Every application, service, API, plugin, and AI capability should make Diana more capable of helping people create, learn, automate, and collaborate.**

This is more specific than the North Star Statement. It adds the **purpose Diana serves**.

### Integration with North Star Statement

**North Star Statement** (Strategic Filter):
> Every capability helps Diana better support people who rely on her.

**Diana's Purpose** (Execution Focus):
> Diana helps people create, learn, automate, and collaborate.

**Combined:**
Every capability should:
1. Help Diana support people (North Star)
2. Enable creation, learning, automation, collaboration (Purpose)

## What This Means for Development

### For Phase 1 (v0.2 Diana Alpha)
- Ask: "Does this make Diana a better conversational assistant?"
- Features: Memory, documents, search, tools, multi-device, multi-model routing
- Not included: Productivity modules, Marketplace, Enterprise features

### For Phase 2a (v0.3 Productivity Suite)
- Ask: "Does this make Diana a better knowledge worker?"
- Features: All productivity modules, automation, AI writer
- Not included: Marketplace, Enterprise features

### For Phase 2b (v0.4 Marketplace)
- Ask: "Does this make Diana a better curator of capabilities?"
- Features: Skill Store, Plugin SDK, Licensing, Billing
- Not included: Enterprise features

### For Phase 3a (v0.5 Enterprise)
- Ask: "Does this make Diana work better for organizations?"
- Features: Teams, Roles, Permissions, SSO, Audit, Analytics
- Not included: Distribution (desktop, voice)

### For Phase 3b (v1.0 AIOS)
- Ask: "Is Diana available everywhere with context preserved?"
- Features: Desktop apps, Mobile, Browser, Voice, Ambient
- Result: Diana is an operating companion

## Decision Quality

With this integrated roadmap, decisions become clearer:

**Question:** Should we build a calendar module?
- ✅ Yes, if it makes Diana a better **productivity partner** (Phase 2b focus, included in v0.3)

**Question:** Should we build an analytics dashboard?
- ✅ Yes, if it helps enterprises measure **Diana's value** (Phase 3a focus, included in v0.5)

**Question:** Should we build skill discovery UI in v0.2?
- ❌ No, skill discovery belongs in v0.4 (Phase 2b), not v0.2. v0.2 focuses on Diana becoming a trusted conversational assistant first.

**Question:** Should we support voice in v0.2?
- ❌ No, voice is a v1.0 (Phase 3b) feature for omnipresence. v0.2 focuses on text-based conversational excellence.

## The Principle That Unifies Everything

**Leadership's Guiding Principle:**
> Every application, service, API, plugin, and AI capability should make Diana more capable of helping people create, learn, automate, and collaborate.

**Apply it daily:**
- When prioritizing features: Does this help Diana **create**?
- When building services: Does this help Diana **learn**?
- When designing automation: Does this help people **automate**?
- When adding collaboration: Does this help people **collaborate**?

If the answer is "no" or "unclear," reconsider.
