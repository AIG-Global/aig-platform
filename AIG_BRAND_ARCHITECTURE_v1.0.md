## Core Definitions

### AIGINVEST
**The destination. The company. The ecosystem.**

- Customer relationship begins here
- Central account and identity
- Cloud platform for all services
- Community and collaboration space
- Marketplace of capabilities
- Learning academy
- Investment platform
- Every Diana user is an AIGINVEST user

### Diana
**The AI companion. Everywhere.**

- Primary interface to AIGINVEST
- Available on web, desktop, mobile
- Available on AIOS devices
- Available in enterprise deployments
- Not tied to any single hardware device
- Unified across all surfaces
- Users interact with Diana primarily

### AIOS
**The operating system. Multiple devices.**

- Runs on North Star ONE
- Runs on future hardware (tablets, mini, vision)
- Can run on desktop/emulator
- Showcases what Diana can do
- Reference implementation of AIGINVEST stack
- Not the only way to access AIGINVEST

### North Star ONE
**The flagship hardware. The reference device.**

- Premium AI smartphone
- Best experience with Diana
- Hardware innovation platform
- Not required to use AIGINVEST
- One of many potential North Star products
- Demonstrates AIOS capabilities

## Core Principle

> **"Every product strengthens AIGINVEST"**

This means:

| Product | Role | Strengthens AIGINVEST By |
|---------|------|-------------------------|
| Diana | AI Companion | Making every surface useful |
| AIOS | Operating System | Showcasing platform potential |
| North Star ONE | Hardware | Premium experience, hardware innovation |
| Marketplace | Extensions | Expanding ecosystem capabilities |
| Academy | Learning | Onboarding users, building community |
| Community | Social | Network effects, stickiness |

## Core Promise

> **"One account. One AI companion. Every device. Every service."**

Breaking this down:

- **One account** → AIGINVEST login
- **One AI companion** → Diana
- **Every device** → Web, phone, desktop, tablet
- **Every service** → Chat, documents, projects, marketplace

This promise is achievable with the architecture we've defined.

---

## What This Changes

### ❌ Old Framing
"Build North Star ONE"  
→ Implies phone-first thinking  
→ Suggests hardware is the destination

### ✅ New Framing
"Build AIGINVEST Platform"  
→ Platform-first thinking  
→ Hardware reinforces platform  
→ Sequenced: Platform → Diana → AIOS → Hardware

## Branding Rules

### DO
✅ "Discover AIGINVEST"  
✅ "Meet Diana on AIGINVEST"  
✅ "AIOS is intelligent OS"  
✅ "North Star ONE is premium experience"  
✅ "Available on all devices"  

### DON'T
❌ "Meet Diana on North Star ONE" (too narrow)  
❌ "AIOS is our ecosystem" (it's the OS)  
❌ "Exclusive phone experience" (contradicts promise)  
❌ "North Star ONE users get Diana" (implies Diana is phone-first)  
❌ "AIGINVEST phone" (AIGINVEST is not the phone)  

---

## Documentation Standards

All future documentation must:

- Use **"AIGINVEST"** when referring to company, platform, ecosystem
- Use **"North Star ONE"** when referring to the hardware device
- Use **"AIOS"** when referring to the operating system
- Use **"Diana"** when referring to the AI companion
- Never position North Star ONE as the ecosystem name
- Always position AIGINVEST as the destination
- Always emphasize platform first, hardware as premium access point

---

## Future Hardware (Without Breaking Brand)

Because we separated the ecosystem from the device, future hardware can scale naturally:

- **North Star ONE** — Phone (premium)
- **North Star Tablet** — Tablet (future)
- **North Star Mini** — Compact device (future)
- **North Star Vision** — AR/VR (future)
- **North Star Home** — Smart home hub (future)

All run AIOS. All access AIGINVEST. All connect to Diana.

All strengthen the core promise: "One account. One AI companion. Every device."

## Architecture Implications for Sprint 1

### Build Sequence
1. ✅ **AIGINVEST Platform** (web, API, cloud services)
2. ✅ **Diana** (primary interface everywhere)
3. ⏳ **AIOS** (operating system, reference device)
4. ⏳ **North Star ONE** (hardware, premium experience)

### Sprint 1 Focus
- AIGINVEST web platform fully functional
- Diana accessible and useful on web
- Conversations and documents saved to cloud
- Community features initialized
- Ready for non-hardware users to explore

### What Sprint 1 Does NOT Need
- Mobile-only features
- Hardware-specific optimizations
- AIOS implementation
- Phone OS customization

## The Competitive Advantage

By separating ecosystem from hardware:

✅ Platform grows independently  
✅ Reduces time to AIGINVEST launch  
✅ Allows faster hardware iteration  
✅ Hardware becomes "premium experience" not "required experience"  
✅ Community forms before hardware arrives  
✅ More users at launch  
✅ Network effects start earlier
