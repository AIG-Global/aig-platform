# docs/adr

Architecture Decision Records (ADRs) - documenting key technical decisions.

## Purpose
ADRs capture important architectural decisions and their rationale, helping the team understand:
- Why certain technologies were chosen
- Trade-offs considered
- Context and consequences
- Alternatives evaluated

## Format
Each ADR follows the template:
- **Title** - Decision name
- **Status** - Proposed/Accepted/Deprecated
- **Context** - Problem and forces
- **Decision** - What was chosen and why
- **Consequences** - Outcomes and trade-offs
- **Alternatives** - Other options considered
