# VISION_GUARD.md

## The Compass

**Read this before contributing.**

This document contains the principles that guide every decision.

When unsure whether to build something, check these principles.

If it doesn't align, it waits.

---

## The Five Principles

### 1. AIGINVEST is the Company and Ecosystem

**Everything serves AIGINVEST.**

Not Diana. Not AIOS. Not the phone.

**AIGINVEST is supreme.**

Diana, AIOS, and North Star ONE exist to strengthen the AIGINVEST ecosystem.

- Every feature must make AIGINVEST stronger
- Every user journey must demonstrate AIGINVEST value
- Every decision must ask: "Does this strengthen the ecosystem?"

### 2. Diana is the Primary Interface

**Diana is the user's first relationship with AIGINVEST.**

Not the website. Not an app. **Diana.**

She's present on:
- The web
- Desktop
- Mobile
- AIOS
- North Star ONE

She's consistent across all devices.

She's the only interface users need.

- Every feature must work naturally with Diana
- Diana must be able to explain or use everything
- No feature exists that Diana can't touch

### 3. AIOS Powers North Star ONE

**AIOS is the operating system.**

**North Star ONE is the flagship AI phone.**

Not separate products. **One integrated experience.**

- AIOS is not an operating system in the traditional sense
- It's the intelligence layer that powers hardware
- North Star ONE demonstrates AIOS at its best

### 4. North Star ONE is the Flagship AI Phone

**The phone is the premium expression of AIGINVEST.**

Not a secondary product. Not an afterthought.

**The flagship device that proves the vision.**

- Every feature AIGINVEST offers should work beautifully on the phone
- The phone is not an app. It's an experience
- Diana is the interface

### 5. Every Feature Must Simplify, Never Complicate

**One company. One account. One AI companion. One seamless experience.**

Not: "One company with 5 products."  
Not: "One account across 10 services."  
Not: "One companion that needs training to use."

**Simpler. Always.**

- Every new feature must reduce complexity for the user
- If it adds buttons, remove others
- If it adds menus, consolidate elsewhere
- Simplicity is the product

## The Measurement

### We Don't Ship Features

We ship **complete user journeys**.

Not: "Authentication system built"  
But: "User can register and login"

Not: "Chat API created"  
But: "User can talk to Diana and get an answer"

Not: "Database layer complete"  
But: "User can close the browser and return tomorrow to find everything restored"

### Every Sprint Answers This

**"Did the user complete one full journey?"**

Not: "How many tickets closed?"  
Not: "How many lines of code?"  
Not: "What features are done?"

But: **"Can a user experience something complete end-to-end?"**

---

## The Decision Framework

### When Someone Suggests a Feature

**Ask:**

1. **Does it strengthen AIGINVEST?** (principle 1)
2. **Can Diana use or explain it naturally?** (principle 2)
3. **Does it simplify the experience?** (principle 5)
4. **Does the user complete a journey?** (measurement)

**All YES:** Build it now.  
**Some YES:** Maybe later.  
**All NO:** Backlog for future.

---

## The Questions That Guide

When building or reviewing, ask:

- Would Diana naturally use this?
- Does a user understand AIGINVEST better after using this?
- Did the user complete something real?
- Does this make the experience simpler?
- If this were on North Star ONE, would it feel premium?
- If we remove this, does the experience break?

## The Discipline

**For every sprint, commit to:**

- One user journey
- Complete end-to-end
- No shortcuts
- Tested thoroughly
- Documented clearly
- Polished UX
- Diana works naturally with it

## If You're Tempted to Add Something Else

**Write it on the backlog.**

Don't:
- Add it to this sprint
- "Just quickly" build it
- Assume you have time
- Compromise the journey

**We finish one thing completely before starting another.**
