## 🔧 Known Issues & Workarounds

### Issue 1: TypeScript Compilation Caching
**Problem:** Compiled dist/main.js contains old ask-diana imports even after source main.ts was updated  
**Impact:** API cannot start, throws ERR_MODULE_NOT_FOUND  
**Workaround:** 
```bash
# Clean and rebuild
rm -r apps/api/dist .turbo node_modules/.pnpm
npm install
npm run build
```
**Root Cause:** Turbo build cache not invalidated properly  
**Solution (Next Session):** 
- Clear .turbo directory before rebuild
- Verify tsconfig include paths don't include old files
- Consider using --force flag with Turbo

### Issue 2: Module Resolution at Runtime
**Problem:** pnpm workspace links (@aig/ aliases) not resolved by ts-node-dev  
**Impact:** Development server fails with module not found  
**Current Status:** Requires build-first approach  
**Next Session:** Implement proper build pipeline with pre-built packages

### Issue 3: ESM Module Type Detection
**Warning:** Node.js warning about module type not specified  
**Fix:** Add `"type": "module"` to package.json (already attempted in some files)
