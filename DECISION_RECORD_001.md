## Working Agreement

We commit to these principles:

### 1. One Source of Truth
All product decisions flow from:
- AIGINVEST Constitution (strategy)
- Daily Question (daily focus)
- Alpha 0.1 definition (what we're shipping)

No ambiguity. No conflicting guidance.

### 2. One Active Milestone at a Time
We ship Alpha 0.1 complete before starting Alpha 0.2.

No parallel work.
No "future proofing."
No hedging bets.

### 3. Nothing is "Done" Until:
- It works (tested)
- It's tested (no edge cases)
- It's documented (clear for next developer)
- It feels polished (users would use it)

**Code quality > delivery speed.**

### 4. Every Sprint Ends with Something Demonstrable
Every week, there's something we can show.

Not "80% done."
Not "mostly working."
Not "internal alpha."

**Demonstrable. Complete. Polished.**

## The Workflow (Starting Tomorrow)

### Morning (15 minutes)

**Standup:**
- Review yesterday's progress
- Choose one high-priority objective
- Identify any blockers
- Confirm commitment

**Example:**
- "Yesterday we fixed authentication flow."
- "Today we're completing database persistence."
- "No blockers. Ready to build."

### During the Day (6+ hours)

**Build that objective:**
- Code the feature
- Test it
- Review architecture if needed
- Ask questions if blocked
- Don't multitask

**One thing at a time.**

### End of Day (30 minutes)

**Demonstrate what works:**
- Show the feature running
- Show the code
- Show it's tested

**Update the roadmap:**
- Mark what's complete
- Update journey progress
- Identify what's next

**Plan the next day:**
- What's the next priority?
- What do we need to unblock?
- What could prevent it?
