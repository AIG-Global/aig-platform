## The Enforcement Mechanism

**Every PR must answer:**

1. Which domain? (Where does it belong?)
2. Which layer? (Infrastructure, Platform, Mission, Diana, Experience)
3. Three rules? (Diana, API, Single Responsibility)
4. Diana integration? (How can Diana use it?)
5. Layer boundaries? (Respect contracts?)
6. Contract defined? (Input/output/error?)
7. Tested? (Unit, integration, E2E?)
8. Observed? (Metrics, logs, alerts?)
9. Secured? (Auth, permissions, audit?)
10. 10-year test? (Still valid at 10x scale?)

**No "unclear" answers allowed.**

**If any is unclear, PR is blocked.**

**This enforces the architecture.**
