### 3. Daily Development Cycle: The 5 Questions

**File:** [DAILY_DEVELOPMENT_CYCLE.md](./DAILY_DEVELOPMENT_CYCLE.md)

1. **Did user experience improve?** (Can they do something new?)
2. **Is code cleaner?** (Better than yesterday?)
3. **Did we reduce complexity?** (Simpler system?)
4. **Is documentation accurate?** (Can someone understand?)
5. **Can we demonstrate it?** (Show working feature?)

**All YES = Good day**  
**Any NO = Fix tomorrow**

**Weekly version (Friday):**

**All YES = On track for August 3**

## If Something Gets Blocked

### Process
1. Identify the blocker
2. Tell founder immediately
3. Founder unblocks same day
4. Agent continues building

### Example
```
Agent: "Can't test without database running."
Founder: "Docker or local PostgreSQL?"
Agent: "Docker running now. Continuing."
```

**We never let blockers stop the priority.**
